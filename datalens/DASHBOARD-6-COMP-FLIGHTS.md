# Дашборд 6: «Сравнение динамики продаж рейсов»

Кратко: в датасете **шесть разных параметров** (не один параметр дважды), в SQL из них собирается XML для процедуры. На дашборде удобнее всего **одна группа селекторов (Group Control)** с шестью полями — одна кнопка «Применить». Легенда и цвета линий совпадают с макетом, если из SQL приходят ожидаемые подписи серий (см. ниже).

## 1. Параметры датасета (имена зафиксируйте такими же в UI)

| Имя параметра | Тип / смысл | Пример значения |
|---------------|-------------|-----------------|
| `cf_nr1` | строка | `ТИ101` |
| `cf_d1` | дата | `2026-04-08` |
| `cf_r1` | строка | `ДМД-НАК` |
| `cf_nr2` | строка | `ТИ104` |
| `cf_d2` | дата | `2026-04-08` |
| `cf_r2` | строка | `НАК-ДМД` |

В редакторе датасета имена должны совпадать с плейсхолдерами в SQL (в YDL/DataLens обычно `{{cf_nr1}}` и т.д. — проверьте подсказку в поле запроса).

## 2. Запрос датасета (шаблон)

Точный вызов (`EXEC` vs `SELECT … FROM fn…`) возьмите из SSMS по объекту `spAZ_Rep_CompFlights`.

Ниже — логика сборки XML как в комментарии начальника:

```sql
DECLARE @xml nvarchar(max) = N'<i><j nr="' + {{cf_nr1}} + N'" d="' + CONVERT(varchar(10), {{cf_d1}}, 23) + N'" r="' + {{cf_r1}} + N'"/><j nr="' + {{cf_nr2}} + N'" d="' + CONVERT(varchar(10), {{cf_d2}}, 23) + N'" r="' + {{cf_r2}} + N'"/></i>';

-- Вариант A: хранимая процедура (уточните имя параметра в SSMS)
EXEC dbo.spAZ_Rep_CompFlights @xml = @xml;

-- Вариант B: если это табличная функция
-- SELECT * FROM dbo.fnAZ_Rep_CompFlights(@xml);
```

Ожидаемая форма даты в XML: `YYYY-MM-DD` (как в примере: `d="2026-04-08"`).

### Колонки результата для линейного графика

Минимум для визуализации «как на макете»:

1. **Ось X** — число «дней до вылета» (целое или число), например `days_before` / `d_before` — как в вашем TVF.
2. **Ось Y** — процент занятости, например `pct` / `load_pct`.
3. **Серия (полка «Цвета»)** — текст для легенды:
   - две линии рейсов: подпись вида **`ТИ101 08.04.2026`** (рейс + дата через пробел, дата `ДД.ММ.ГГГГ` — как на скрине);
   - три пороговые линии: **`60% загрузка`**, **`80% загрузка`**, **`100% загрузка`** (важно слово «загрузка» и проценты — под это завязаны цвет и растягивание линий на оси X в UI).

Пример (псевдо-SQL внутри обёртки над процедурой):

```sql
SELECT
    days_before,
    load_pct,
    series_title  -- значения: 'ТИ101 08.04.2026', 'ТИ104 08.04.2026', '60% загрузка', ...
FROM ...
```

Если процедура отдаёт «широкую» таблицу без столбца серии — добавьте в датасет `UNION ALL` / `UNPIVOT`, чтобы получить длинный формат `(x, y, series_title)`.

## 3. Что сделать в UI (воркбук / дашборд)

1. **Датасет** — создать по запросу выше, объявить **6 параметров** с именами из таблицы.
2. **Чарт** — тип **Линейная диаграмма**:
   - **X** — дни до вылета;
   - **Y** — процент занятости;
   - **Цвета** (или измерение, по которому строятся серии) — поле `series_title` / аналог.
3. **Дашборд, вкладка 6**:
   - Добавить **группу селекторов** (Group Control), внутри **6 элементов**: Дата1, Рейс1, Напр-е1, Дата2, Рейс2, Напр-е2.
   - Каждый элемент привязать к **своему** параметру (`cf_d1` … `cf_r2`). Так нет конфликта «один параметр дважды» и одна кнопка применения на группу.
4. Положить на вкладку виджет с чартом; связать чарт с тем же датасетом / параметрами вкладки (как у остальных отчётов Repka).

## 4. Легенда и цвета (уже в коде платформы)

Файл: [`../components/datalens-ui/src/server/modes/charts/plugins/datalens/preparers/line/gravity-charts.ts`](../components/datalens-ui/src/server/modes/charts/plugins/datalens/preparers/line/gravity-charts.ts)

- Пороги **`60% загрузка`**, **`80% загрузка`**, **`100% загрузка`** (и близкие варианты подписи) получают **`#ff0000`**, **`#ffff00`**, **`#2f972f`**.
- Если в данных ровно **2 серии рейсов** (подписи без «% загрузки») и **3 пороговые** — первые две серии по алфавиту получают **`#ff00ff`** и **`#972f97`** (как два рейса на макете), легенда — те строки, что вернул SQL.
- Дополнительно можно в SQL использовать технические ключи **`FLIGHT_1`**, **`FLIGHT_2`**, **`REF_60`**, **`REF_80`**, **`REF_100`** на полке цветов — для них заданы те же hex.

Итог: **в легенде будет то, что вы отдаёте в поле серии** (например `ТИ101 08.04.2026`), а цвет линии и маркер в легенде совпадают, потому что цвет задаётся на серию в том же preparer.

## 5. Ограничение

Полное «как на скрине» возможно только если TVF/процедура отдаёт данные по **каждому** дню до вылета для обоих рейсов и трёх горизонталей; иначе нужно доработать SQL на стороне БД.
