# Обновление данных дашборда только по кнопке «Применить»

Чтобы фильтры (Дата, Группа, Рейсы и т.д.) **не** обновляли чарты при каждом изменении, а данные перезапрашивались **только после нажатия «Применить»**, в DataLens нужно включить соответствующую опцию у группы селекторов.

## Как включить

1. Откройте дашборд в **режиме редактирования** (кнопка «Редактировать» или иконка карандаша).
2. Выберите **блок с фильтрами** (группу, в которой находятся Дата, Группа, Рейсы) — клик по области селекторов.
3. Откройте **расширенные настройки** этой группы:
   - через меню элемента (три точки или шестерёнка у группы),  
   - или через пункт «Настройки» / «Расширенные настройки» в зависимости от версии UI.
4. В разделе **«Параметры группы»** включите чекбокс **«Кнопка «Применить»»** (или «Apply button» в англоязычном интерфейсе).
5. Сохраните дашборд.

После этого:

- Кнопки **«Применить»** и при необходимости **«Сбросить»** появятся рядом с фильтрами.
- Изменение значений в селекторах (дата, группа, рейсы) **не** будет сразу отправлять запрос к данным.
- Запрос с новыми параметрами уйдёт **только после нажатия «Применить»** — тогда обновятся связанные виджеты (в т.ч. сводная таблица).

## Дополнительно: зависимые селекторы

Если в группе **несколько** селекторов (например, «Группа» и «Рейсы») и включена кнопка «Применить»:

- В расширенных настройках группы может отображаться опция **«Обновлять зависимые селекторы до нажатия «Применить»»**.
- При её включении списки в зависимых селекторах (например, список рейсов в зависимости от выбранной группы) обновляются сразу при выборе значений, но **виджеты с данными** по-прежнему обновляются только после нажатия «Применить».

Так вы получаете логику: **параметры и новый запрос применяются только после клика по «Применить»**.
