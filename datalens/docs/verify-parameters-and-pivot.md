# Проверка в браузере: параметры датасета и сводная таблица

Пошаговая проверка, что параметры подставляются в запрос датасета и сводная таблица открывается без ошибок.

## Пересборка UI (если менялся код интерфейса)

Чтобы применились правки в коде DataLens UI (подстановка значений параметров при превью):

1. Перейдите в каталог UI: `cd components/datalens-ui` (от корня репозитория).
2. Соберите проект: `npm run build`.
3. Если DataLens поднимается через Docker и использует локальную сборку — пересоберите образ или смонтируйте новый `build` в контейнер по инструкции вашего окружения.
4. Перезапустите контейнер UI (например, `docker compose restart ui` в каталоге `datalens`).

Бэкенд (data-api) перезапускать не нужно для этих изменений.

## 1. Датасет «Репка»: параметры и запрос

1. Откройте воркбук **Сводная**: `http://localhost:8080/workbooks/503xbcpj4flmp`.
2. Перейдите на вкладку **Датасеты** (или откройте `?tab=dataset` в том же воркбуке).
3. Откройте датасет **Репка**.
4. Проверьте:
   - Включите **параметризацию** в настройках датасета (иконка шестерёнки вверху справа) — тогда появится возможность «Использование в настройке источника». У каждого параметра в этой колонке должно быть **Разрешено**; если «Запрещено» — откройте параметр (Редактировать), включите **«Разрешить использовать в настройке источника»**, сохраните.
   - **Значение по умолчанию**: если при сохранении появляется ошибка `PARAMETER_VALUE_INVALID`, оставьте значение по умолчанию пустым в датасете и задайте значения в чарте или через фильтры дашборда (см. `dataset-tvf-parameters.md`, раздел про PARAMETER_VALUE_INVALID). Иначе задайте значение по умолчанию, чтобы превью и чарт могли выполнить запрос.
   - Во вкладке **Источники** в подзапросе (Subselect) запрос вида:
     ```sql
     select * from [dbo].[fnAZ_Rep_MainW](
       {{<id_параметра_даты>}},
       {{<id_параметра_рейсов>}},
       null, null
     )
     ```
     Здесь должны стоять **реальные id (guid)** параметров, не имена. Id можно взять через **Скопировать ID** в меню параметра в списке полей.
5. Сохраните датасет, если вносили изменения.

## 2. Копирование ID параметра

1. В датасете на вкладке **Датасет** в списке полей найдите параметр (иконка/тип «Параметр»).
2. Откройте меню строки (три точки) → **Скопировать ID**.
3. Должно появиться уведомление об успешном копировании. В буфере обмена будет **id параметра** (guid или имя, если бэкенд отдаёт имя как id). Этот же идентификатор нужно использовать в запросе в виде `{{id}}`.

## 3. Сводная таблица в мастере чартов

1. Откройте чарт **Репка — Сводная таблица**: из воркбука перейдите в **Чарты** и откройте этот чарт (или по прямой ссылке, если известна).
2. Должны загрузиться данные и отобразиться сводная таблица (столбцы, строки, показатели). Если раньше была ошибка «Data loading error» из‑за типов text/ntext в MSSQL, после применения патча `query_compiler.py` и перезапуска **data-api** ошибка должна пропасть.
3. Если видите сообщение об ошибке загрузки данных — проверьте логи **data-api** и что патч MSSQL подключён (см. `wizard-pivot-mssql-error.md`).

## 4. Дашборд и фильтры (параметры)

1. Откройте дашборд **Сводная** (из того же воркбука, вкладка **Дашборды**).
2. Убедитесь, что элементы управления (календарь/месяц, группа, рейсы) привязаны к **параметрам датасета** в настройках виджетов.
3. Включите при необходимости **кнопку «Применить»** у группы фильтров, чтобы данные обновлялись только после нажатия «Применить» (см. `dashboard-apply-button.md`).
4. Выберите период, группу или рейсы и нажмите **Применить**. Сводная и другие чарты должны перезапросить данные с новыми значениями параметров.

## Итог

- **Параметры используются**: в запросе датасета стоят плейсхолдеры `{{id_параметра}}` с реальными guid параметров; значения приходят из фильтров дашборда или значений по умолчанию.
- **Сводная появляется**: чарт «Репка — Сводная таблица» и дашборд открываются без ошибки загрузки данных; при изменении фильтров и нажатии «Применить» данные обновляются.

Если на каком-то шаге возникает ошибка — см. `dataset-tvf-parameters.md`, `wizard-pivot-mssql-error.md`, `dashboard-apply-button.md`.
