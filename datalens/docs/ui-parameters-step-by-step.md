# Пошаговая настройка параметров датасета и TVF в UI

> Правильный порядок действий, чтобы датасет с параметрами `dte` и `nrs` сохранялся, превью работало, а на дашборде подставлялись значения.

---

## Перед началом (один раз)

1. **Запуск только из каталога `datalens/`**  
   Иначе подхват патчей не сработает. В PowerShell:
   ```powershell
   cd "d:\YDL OS\datalens"
   docker compose up -d
   docker compose restart control-api data-api
   ```
2. **Проверка, что патч применён** (если при сохранении параметров снова ошибка):
   ```powershell
   docker exec datalens-control-api-dev grep "accept anyway" /tmp/dl_core_ydl/components/accessor.py
   ```
   Должны появиться две строки с `pass  # accept anyway`. Если нет — пересоберите: `docker compose up -d --force-recreate control-api data-api` из каталога `datalens/`.
3. Откройте UI: **http://localhost:8080**. После перезапуска бэкенда обновите страницу (F5 или Ctrl+F5).

---

## Шаг 1: Параметры датасета (вкладка «Параметры»)

1. Откройте датасет **Repka** (или создайте новый).
2. Перейдите на вкладку **«Параметры»**.
3. Добавьте два параметра (если ещё нет):
   - **dte** — тип **Строка**, значение по умолчанию: `01/09/2024`
   - **nrs** — тип **Строка**, значение по умолчанию: `B2731,B2732` (можно и с пробелами после запятых; в SQL всегда подставится одна строка в кавычках: `'B2731,B2732'`)
4. **Пока не включайте** «Использование в настройке источника» — оставьте **Запрещено**.
5. Нажмите **«Сохранить»** (сохранить датасет).  
   — Если появится ошибка про невалидное значение по умолчанию, значит бэкенд не с патчами: перезапустите из `datalens/`:  
   `docker compose restart control-api data-api`.

---

## Шаг 2: Включить параметризацию датасета (шестерёнка)

1. Рядом с названием датасета нажмите **шестерёнку** (настройки датасета).
2. Включите опцию **«Параметризация»** / **«Разрешить использование параметров в источниках»** (если есть такая на уровне датасета).
3. Сохраните настройки.

---

## Шаг 3: Разрешить использование параметров в источнике

1. Снова откройте вкладку **«Параметры»**.
2. Для параметра **dte**: нажмите на строку или на иконку настроек и включите **«Использование в настройке источника»** (Разрешить).
3. То же для параметра **nrs**.
4. Нажмите **«Сохранить»**.  
   — С текущими патчами бэкенда сохранение должно пройти без ошибки `PARAMETER_VALUE_INVALID`.

---

## Шаг 4: Источник с подзапросом (вкладка «Источники»)

1. Перейдите на вкладку **«Источники»**.
2. Выберите источник (например **Repka**) и откройте его настройки (или добавьте источник типа **Подзапрос** / **SQL**).
3. Тип источника: **Подзапрос** (MSSQL Subselect).
4. В поле **«Подзапрос»** введите:
   ```sql
   select * from [dbo].[fnAZ_Rep_MainW]({{dte}}, {{nrs}}, null, null)
   ```
   Имена в `{{...}}` должны **совпадать с id параметров** (у вас оба — `dte` и `nrs`).
5. Название источника можно оставить **Repka**.
6. Нажмите **«Применить»** в диалоге источника.  
   — В запрос подставятся значения по умолчанию (`01/09/2024`, `B2731, B2732`), превью должно отработать без ошибки «compile errors».
7. Закройте диалог и нажмите **«Сохранить»** датасет (если ещё не сохраняли).

---

## Шаг 5: Поля и превью

1. Перейдите на вкладку **«Поля»** и убедитесь, что поля датасета подтянулись (если источник вернул колонки).
2. При необходимости обновите **«Предпросмотр»** внизу — данные должны отображаться с подставленными значениями по умолчанию.

---

## Шаг 6: Дашборд и фильтры

1. Создайте или откройте дашборд.
2. Добавьте чарт на основе датасета **Repka** (сводная таблица).
3. Добавьте три фильтра и **обязательно привяжите каждый к параметрам датасета Repka** (см. ниже).
4. **Дата** — виджет «Диапазон дат» или «Месяц» → в настройках фильтра укажите **привязку к параметру датасета**: параметр **dte** датасета **Repka**. Формат даты для TVF лучше задавать **YYYY-MM-DD** (например `2024-09-01`), если в источнике «нет данных» при формате 01/09/2024.
5. **Группа** — источник AZ_FLIGHTS_GROUPS; при необходимости привяжите к параметру датасета Repka (если группа влияет на запрос).
6. **Рейсы** — источник AZ_FLIGHTSinGROUPS → привязка к параметру **nrs** датасета **Repka** (значения в запрос передаются одной строкой в кавычках, например `'B2731,B2732'`).
7. После выбора значений нажмите **«Применить»** — запрос к БД уйдёт с выбранными dte и nrs; в сводной таблице появятся данные по выбранному месяцу.

**Важно:** если фильтры **не привязаны** к параметрам датасета Repka, при нажатии «Применить» бэкенд не получит новые значения и подставит в запрос только значения по умолчанию — таблица останется «по умолчанию» или пустой. В настройках каждого фильтра должна быть указана связь с параметром (dte или nrs) **именно датасета Repka**, на котором построен чарт.

---

## Варианты для фильтров «Группа» и «Рейсы» — строго из справочников

Чтобы пользователь выбирал только значения из таблиц-справочников (а не вводил произвольный текст):

1. **Два отдельных датасета под фильтры:**
   - **Датасет «Группы»** — источник таблица **AZ_FLIGHTS_GROUPS** (поля групп: код, название и т.п.).
   - **Датасет «Рейсы в группах»** — источник таблица **AZ_FLIGHTSinGROUPS** (связка группа–рейс; поле с кодом рейса и при необходимости с группой).

2. **В настройках виджета фильтра «Группа»:**
   - Тип: список (выпадающий список или мультиселект).
   - **Источник списка:** «Из датасета» → выберите датасет на **AZ_FLIGHTS_GROUPS**, укажите поле с группой (название или код). Варианты фильтра будут подгружаться **только из этой таблицы**.
   - Привязка к параметру датасета Repka: если у Repka есть отдельный параметр под группу — привяжите к нему; если группа влияет только на список рейсов (каскад), настройте зависимость фильтра «Рейсы» от «Группы».

3. **В настройках виджета фильтра «Рейсы»:**
   - Тип: мультиселект (список с множественным выбором).
   - **Источник списка:** «Из датасета» → выберите датасет на **AZ_FLIGHTSinGROUPS**, укажите поле с кодом рейса. Варианты будут **строго из справочника**.
   - **Привязка к параметру:** параметр **nrs** датасета **Repka**. Выбранные рейсы (одна или несколько) передадутся в запрос как одна строка в кавычках, например `'B2731,B2732'`.

Так фильтры «Группа» и «Рейсы» работают только со значениями из AZ_FLIGHTS_GROUPS и AZ_FLIGHTSinGROUPS; в TVF попадают только выбранные из справочников коды.

---

## Итоговая схема

- **Три датасета** (в т.ч. Repka с TVF и параметрами dte, nrs).
- **Итоговый дашборд:** фильтры «Дата» (месяц), «Группа» (AZ_FLIGHTS_GROUPS), «Рейсы» (AZ_FLIGHTSinGROUPS) → кнопка **Применить** → сводная таблица по датасету Repka показывает данные по выбранному месяцу.

---

## Если что-то пошло не так

| Проблема | Действие |
|----------|----------|
| Ошибка «Default value for field nrs/dte is not valid» при сохранении | Запуск через `.\run-own-images.ps1` (образы aeronavigatorbi). После изменений в backend: пересборка `.\build-all-images.ps1` и перезапуск или `docker compose -f docker-compose.yaml -f docker-compose.own-images.yaml restart control-api data-api`. |
| В превью источника ошибка «compile errors» / плейсхолдеры не подставлены | Проверьте, что для **dte** и **nrs** включено «Использование в настройке источника» и датасет сохранён. Проверьте, что в подзапросе указаны `{{dte}}` и `{{nrs}}` (id параметров). |
| ERR.DS_API.DB.COLUMN_DOES_NOT_EXIST / Invalid column name 'B2731' | Раньше значения подставлялись в SQL без кавычек. В YDL OS исправлено: строки и даты подставляются как SQL-литералы (`'01/09/2024'`, `'B2731, B2732'`). Перезапустите control-api и data-api после обновления патчей: `docker compose up -d --force-recreate control-api data-api` из каталога `datalens/`. |
| «Key dte not found in {}» | Включите параметризацию датасета (шестерёнка) и «Использование в настройке источника» для обоих параметров, затем сохраните. |
| После «Применить» показываются только данные по умолчанию (или пустая таблица) | Фильтры не передают значения в запрос. В настройках **каждого** фильтра укажите **привязку к параметру датасета**: дата → параметр **dte** датасета **Repka**, рейсы → параметр **nrs** датасета **Repka**. Чарт должен быть построен на датасете Repka. Формат даты для dte — лучше **YYYY-MM-DD** (например `2024-09-01`). |
| В сводной отображаются только те дни месяца, по которым есть данные; нужны все дни 1…31 | Сводная строит столбцы только по тем значениям измерения, которые пришли в данных. Чтобы всегда были **все числа месяца** (и дни недели), источник данных (TVF или запрос в MSSQL) должен возвращать строки **на каждый день месяца**; для дней без рейсов — с NULL по показателям. См. [BACKEND-CALENDAR-AND-DATE-FILTER-SPEC.md](../../docs/BACKEND-CALENDAR-AND-DATE-FILTER-SPEC.md) (раздел «Как добиться отображения всех дней месяца»). |

Подробнее: [dataset-tvf-parameters.md](dataset-tvf-parameters.md).
